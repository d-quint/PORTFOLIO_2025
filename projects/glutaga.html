Glutaga

A Galaga-inspired 2D space shooter built from scratch with C++ and OpenGL/GLUT, showcasing low-level graphics programming, custom pixel-art rendering, and GPU-accelerated particle systems.

Overview

In the distant future, humanity's last hope rests with you - the pilot of the experimental GLUTAGA starfighter. Earth's resources have been depleted, and mysterious alien forces guard the path to habitable worlds. As Earth's most skilled pilot, you must navigate through waves of alien defenders, clearing the way for humanity's survival ships that follow in your wake.

Glutaga is a complete game engine built from the ground up using C++ and OpenGL/GLUT, developed as an academic project to demonstrate mastery of low-level graphics programming and game architecture. Every visual element—from the player's rotating starfighter to the explosion particle systems—is rendered using custom vertex buffer objects (VBOs) and GPU-accelerated drawing primitives. The game features a component-based architecture with separate modules for player control, enemy AI, projectile physics, particle effects, parallax backgrounds, and UI rendering.

The GLUTAGA starfighter represents a technical showcase of OpenGL's vertex attribute arrays, where custom pixel-art sprites are defined in text files and loaded into GPU memory at runtime. The game implements smooth physics-based movement with acceleration and friction, collision detection using axis-aligned bounding boxes, dynamic color interpolation for visual feedback, and a persistent high score system. All rendering is handled through immediate mode OpenGL commands, demonstrating fundamental understanding of graphics pipeline operations without relying on modern abstraction layers.

Key Features

Custom OpenGL Graphics Engine

Built entirely from scratch using OpenGL/GLUT with vertex buffer objects (VBOs) for GPU-accelerated rendering. Custom sprite loader reads pixel-art coordinates from text files and uploads vertex/color data directly to GPU memory, demonstrating low-level graphics programming without game engine dependencies.

GPU-Accelerated Particle Systems

Real-time particle effects for explosions, damage, and engine trails using custom ParticleSystem and Particle classes. Features physics-based particle motion with velocity, lifespan, and size interpolation, all rendered efficiently through batched OpenGL draw calls with dynamic alpha blending.

Physics-Based Movement System

Smooth acceleration-based movement with friction dampening and boundary collision detection using axis-aligned bounding box (AABB) algorithms. Player starfighter features momentum-based controls, wall bounce physics with configurable restitution coefficients, and precise projectile-enemy collision detection at 60 FPS.

Parallax Scrolling Space Environment

Multi-layered parallax background system featuring procedurally positioned stars and planets that scroll at different speeds to create depth illusion. Planet class implements random size/color generation with circular primitive rendering, while Star class manages twinkling effects and seamless vertical wrapping.

Wave-Based Enemy AI System

Object pooling architecture for efficient enemy management with sine-wave horizontal movement patterns and randomized spawn timing. Enemies feature independent projectile systems, death animations with particle explosions, and dynamic respawning logic, all managed through collision detection callbacks and state machines.

Persistent High Score & Game Over System

UI system with file I/O for saving high scores between sessions, displaying real-time health bars, score counters, and game over screens. Implements callback-based event system for player death sequences, smooth screen transitions, and restart functionality with complete game state resets.
